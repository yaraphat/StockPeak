## Vision

### 10x Check

The 10x version of StockNow isn't a stock picks newsletter. It's a personal AI trading desk for every DSE retail investor. Imagine: you open your phone at 6:55 AM, and StockNow has already analyzed 400 stocks, selected 3, told you exactly when to buy, when to sell, and whether today is even a good day to trade. During market hours, it watches your positions and alerts you when to exit. On Saturday, it shows you how your week went and what you should learn from it. All in Bengali. For $3/month.

That's what we're building toward. The launch version is the first layer: morning picks + exit alerts + weekly report + performance transparency.

## Architecture Decision

**Approach B: Next.js Monolith** chosen over Full Stack Split or Python Monolith.

- Next.js 14 handles frontend + API routes + auth (NextAuth.js)
- Python cron script (standalone) handles daily picks + email + Telegram delivery
- Python intraday script handles exit alert monitoring during market hours
- PostgreSQL (Supabase/Neon) for all data
- ML service added as separate FastAPI service in Phase 2
- Single Vercel deployment for web app

## Failure Modes

| Failure | Fallback |
|---------|----------|
| DSE scrape fails (6 AM) | Retry 3x, fallback scraper, then "no picks today" notification |
| Claude API down | GPT-4o fallback. If both fail: raw technical screen without LLM reasoning |
| Zero stocks pass screen | Relax to 3-of-5 scoring. If <3 pass: "low signal day" with market mood only |
| Intraday polling fails | Log error, retry next 15-min window. Alert admin if 3 consecutive failures |
| Email delivery failure | Telegram as parallel channel. Retry queue for email |
| SSLCommerz down | Show "payment temporarily unavailable" page. Free trial still works |

## Data Source Risk

**DSE has no public real-time API.** This is the #1 technical risk.

- **EOD data (daily picks):** bdshare Python package (primary), direct dsebd.org scraping (fallback), manual CSV entry (emergency)
- **Intraday data (exit alerts):** Poll dsebd.org market page every 15 min during hours (10:00-14:30 UTC+6). This is fragile. Mitigation: test for 2 weeks before enabling exit alerts for users. If scraping breaks, disable exit alerts temporarily and notify admin.
- **Future:** DSE iMDS license (Phase 2) for official delayed data feed
