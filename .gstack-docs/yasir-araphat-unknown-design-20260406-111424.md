## Problem Statement

Bangladeshi retail investors on the Dhaka Stock Exchange (DSE) currently cobble together Facebook groups, Telegram tip channels, manual DSE website checking, and broker phone calls to make investment decisions. This fragmented workflow costs them time they don't have (they're service professionals with demanding day jobs) and money (bad trades from unverified tips, emotional decisions, no technical analysis).

StockLens BD proved the market exists at ৳299/month (~$2.70 USD) with GPT-4o-powered daily picks. StockNow enters this validated market with a better product, better delivery (web + email + Telegram), and a path to custom ML differentiation.

## Demand Evidence

- A direct competitor (StockLens BD) has paying subscribers at ৳299/month, proving willingness to pay for AI stock analysis on DSE
- Founder knows specific service professionals (doctors, engineers, teachers) who actively struggle with the current fragmented workflow
- The status quo is a mix of unreliable Facebook/Telegram tips, manual DSE website checking, and broker dependency — high effort, low reliability
- Target user archetype: Dhaka-based professional with ৳5-20 lakh invested, trading part-time, price-insensitive at ৳299-500/month if it saves time and improves outcomes

## Status Quo

The typical DSE retail investor's daily workflow:
1. Check 2-3 Facebook groups for stock tips during morning commute
2. Open dsebd.org to check yesterday's prices and today's pre-market
3. Maybe check a Telegram channel for "expert" picks
4. Call broker or check broker app for recommendations
5. Make a buy/sell decision based on partial, unverified information
6. Track portfolio in a personal spreadsheet or mental math

This costs ~30-60 minutes daily of fragmented attention, produces decisions based on rumors rather than data, and results in predictable mistakes: no stop-losses, emotional trading, insufficient diversification, and following market sentiment without verification.

## Target User & Narrowest Wedge

**Target user:** Service professional in Dhaka (doctor, engineer, teacher). ৳5-20 lakh invested in DSE. Trades 2-5 times per week around a demanding day job. Educated enough to want data-driven decisions. Gets most stock information from unstructured social media. Has been burned by a bad tip in the last 6 months.

**Narrowest wedge:** 3 AI-analyzed stock picks delivered daily before market open (9:30 AM BST), with buy zone, target price, stop-loss, confidence score, and 1-paragraph reasoning. Delivered via:
- Email (7:00 AM)
- Telegram bot (7:00 AM)
- Web dashboard (always available)

This is the smallest version someone would pay ৳299/month for. Everything else (portfolio tracker, screener, real-time data, chatbot) is expansion.

## Constraints

1. **Data access:** DSE data available via scraping (dsebd.org, bdshare) for delayed/EOD data. Real-time data requires DSE iMDS license (Phase 2)
2. **Regulatory:** Ship as "educational/informational AI analysis" with prominent disclaimers. Begin BSEC Investment Adviser registration in parallel (6-12 months)
3. **Payment:** Bangladesh market requires local payment (bKash, Nagad, SSLCommerz). International cards are rare among retail investors
4. **Language:** Primary audience reads Bengali. UI should support Bengali with English as secondary. Use `next-intl` for i18n. At launch: UI chrome in Bengali, pick reasoning generated in Bengali directly via LLM prompt (not translated). English available as toggle.
5. **Trust:** New platform competing against established StockLens BD. Track record transparency is critical
6. **Privacy:** Comply with Bangladesh Digital Security Act. Add `/privacy` page. Collect minimal PII (email, phone for bKash). No data sharing with third parties.
7. **Anti-abuse:** Email verification required for account creation. Rate-limit signups by IP. Prevents free-tier multi-account abuse.

## Premises

1. **Daily AI picks = the core wedge.** Everything else is expansion. The product is "tell me what to buy before market opens."
2. **Simplified architecture for launch.** Next.js + Python backend + PostgreSQL. No Kafka, Go gateway, QuestDB, or message bus until 10,000+ users.
3. **Regulatory compliance runs in parallel.** Ship with disclaimers, register with BSEC alongside. StockLens BD operates the same way.

### Daily Pick Generation Pipeline

All times in BST (UTC+6):

1. **6:00 AM** — Python script scrapes yesterday's EOD data from DSE (all ~400 listed stocks). Primary source: bdshare package. Fallback: direct dsebd.org scraper. If both fail: retry 3x with 2-min backoff, then alert via Telegram to operator and send "no picks today — data unavailable" to subscribers.
2. **6:10 AM** — Technical screen filters to top 15-20 candidates using AND logic (all criteria must pass):
   - Volume spike (>1.5x 20-day average)
   - RSI between 30-60 (not overbought)
   - Price above 50-day SMA
   - Positive MACD crossover or approaching
   - Sector momentum alignment
   If fewer than 10 stocks pass all filters, relax to 3-of-5 scoring (weighted). If fewer than 3 pass any combination, flag as "low-signal day" and note it in the pick delivery.
3. **6:15 AM** — LLM (Claude API, fallback: GPT-4o) analyzes top candidates with prompt:
   - Technical indicators summary
   - Recent price action context
   - Sector/market context
   - Output: structured JSON with 3 picks, each containing: ticker, buy_zone, target, stop_loss, confidence (1-10), conviction_modifier (-1 to +2), and reasoning (Bengali)
   - **6:20 AM** — Validation step before storage:
     - Verify ticker exists in scraped dataset
     - Check stop_loss < buy_zone < target
     - Check confidence is integer 1-10
     - If any pick fails validation, re-prompt LLM for that slot (max 2 retries)
4. **6:30 AM** — Store validated picks in PostgreSQL
5. **7:00 AM** — Deliver via email + Telegram

**Confidence score definition:** 1-10 integer. Calculated as: number of technical indicators aligned (0-5) + sector momentum score (0-3, where sector momentum = sector index 5-day return > 0 scores 1, > DSE broad index scores +1, top-3 sector scores +1) + LLM conviction modifier (-1 to +2, returned as structured JSON field: -1 = counter-signals present, 0 = neutral, +1 = strong narrative catalyst, +2 = multiple converging signals). Displayed to users as a simple 1-10 rating with label (1-3: Speculative, 4-6: Moderate, 7-10: High Confidence).

### Telegram Bot Architecture

- **Public channel** (@StockNowBD): Broadcast-only. Posts 1 free pick daily at 7:00 AM (delayed, no reasoning). Anyone can join.
- **Bot** (@StockNowBot): Interactive. Commands: `/today` (today's picks, Pro only), `/history` (last 7 days), `/subscribe` (link to web payment), `/help`.
- **Account linking:** After web signup, user gets a one-time 6-digit token. Send token to bot via `/link TOKEN` to connect Telegram account to StockNow Pro subscription. Bot then sends full 3 picks with reasoning via DM at 7:00 AM.

## Failure Modes

| Failure | Impact | Mitigation |
|---------|--------|------------|
| DSE scraper down | No picks generated | Dual source (bdshare + direct scraper), 3x retry with backoff, "no picks today" notification |
| LLM API down | No analysis | Claude primary, GPT-4o fallback. If both fail: send raw technical screen results without LLM reasoning |
| Email delivery failure | Subscribers don't get picks | Telegram as parallel delivery. Email retry queue. Dashboard always shows picks |
| Payment gateway down | Can't collect revenue | SSLCommerz primary, bKash secondary. Free tier doesn't require payment |
| Database down | No pick storage/retrieval | Managed PostgreSQL (Supabase/Neon) with automated backups. Picks also cached in-memory for Telegram/email delivery |
